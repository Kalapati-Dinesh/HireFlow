## What caused the 403 and what was fixed

**Root cause 1 — Missing routes in `permitAll()`**

The `SecurityConfig` only allowed `/api/login`, `/login`, `/`, `/css/**`, `/js/**`, and Swagger. Every other URL — including `/signup`, `/register`, `/logout`, and all the MVC candidate routes — was blocked by Spring Security before the request even reached the controller. That's why you got 403.

Added all missing routes to `permitAll()`:
- `/signup`, `/register`, `/logout`
- `/addCandidate`, `/saveCandidate`, `/viewCandidates`, `/updateStatus`, `/searchCandidate`, `/filterStatus`
- `/api/refresh`

Also added `.anyRequest().authenticated()` at the end so unmatched routes are properly secured instead of silently denied.

**Root cause 2 — `STATELESS` session conflicting with MVC `HttpSession`**

The session policy was set to `STATELESS` (meant for pure REST/JWT APIs), but the Thymeleaf MVC controllers use `HttpSession` to track logged-in users. With `STATELESS`, Spring Security destroys the session on every request, so `session.getAttribute("loggedInUser")` always returns null and every page redirects to `/login`.

Changed to `IF_REQUIRED` — sessions are created only when needed (MVC flow), while JWT Bearer token auth still works normally for the REST API.
